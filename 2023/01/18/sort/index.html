算法分析

采用归并排序（MergeSort）的方式完成此题。

考虑compare的调用次数\(T(n)\)。假如我们能做到k路归并，那么有\(T(n) = k * T(n/k)+O(n),T(k)= 1\)。简单分析一下：

\(T(n) = k * T(n/k) + n\)

​ \(= k*(k * T(n/k^2) + n / k) + n\)

​ $ ... $

​ $ = k^mT(n/k^m) + m n$

其中\(m = log_kn\)，故\(T(n) = n + nlog_kn \approx nlog_kn\)

对于此题，若k = 3，\(T(n)\approx 1.2*10^7\)，超出范围。

若k = 4，\(T(n)\approx 9.96*10^6\)，恰好满足。

这便要求我们根据compare带来的信息完成四路归并。简单来说，在归并时我们先任取三路进行一次compare。由于每次归并都是将最小的元素加入到数组中，所以三路compare中的最大者一定不会在这一次，以及下一次合并数组操作中被加入到数组中去（因为min和mid都比他要小）。

故我们每次compare时，得到的最大者需进入”等待“状态。例如某一次compare数组A[], B[], C[]中的首元素，且C[]是最大者。那么在下次归并时，我们通过compare(A[], B[], D[])来挑选下一个被合并到数组的元素，而无需考虑C[]。算法的正确性是由上述的事实保证：C[]不仅比最小者小，还比中间者小。

从另一个角度想，如果我们仅仅采用三路归并的方式，那么compare返回的最大值信息被我们忽略了，相当于没有充分地利用所得到的信息。对于这题来说，compare三路元素恰好能使我们完成四路归并。要想完成更多路的归并，需要使compare一次能比较的元素增加。

四路归并 -> 三路归并 -> 二路归并

在归并过程中，总会有一个数组先被消耗完。这里采用的策略是统一将数组转移到A[], B[], C[]或A[], B[]中。这减少了代码的重用率。

复杂度分析

时间复杂度

由前，时间复杂度也满足\(T(n) = k * T(n/k)+O(n)\)的递推式。故复杂度为\(O(nlog_kn)\)。（然而实际上compare函数不是真的能在一个时间单元内完成，相比于普通的二路比较）

空间复杂度

空间消耗来自归并时的辅助数组。与二路归并的渐进复杂度一致，空间复杂度为\(O(n)\)。
