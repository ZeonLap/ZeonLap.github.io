算法分析

Segment Tree

由于这题点数\(n\)的范围很大（\(1\le n<2^{31}\)），而总操作数\(m\le200000\)远小于\(n\)。故这题可以采用离散化的方式，先将\(m\)次操作全部记录下来，这至多涉及\(2m\)个点。将他们排序去重后，假设这些点与\(0、n\)组成有序的\(k\)个点，那么这\(k\)个点同时也构成了\(k-1\)个区间。在接下来的操作中，我们关心的只是该操作对应到每个区间的影响——也即线段树的处理方式。

具体来说，为了使区间之间不重叠，我们将输入中的所有闭区间\([L,R]\)转变为开区间\((L-1, R]\)。将所有区间端点存下来后，我们再加入两个端点\(0\)与\(n\)。将他们排序去重，得到端点集points[]。端点集合中的\(k\)个端点（包括\(0\)与\(n\)）构成了\(k-1\)个区间，其中第\(i\)个区间的左右端点分别为points[i]和points[i+1]。

于是，一棵以区间为结点的线段树就已经建好了（线段树的知识在此不赘述）。其叶子结点对应极小区间（不能再被分割），两个结点的父结点对应他们两个区间的并（例如\((2,4]\)与\((4,6]\)的父节点代表\((2,6]\)）。每个结点的值代表了该区间一共被翻转了几次。

在程序中，线段树以long long segTree[seg_size << 2]储存，其中seg_size代表区间总数。为了方便下面说明，先展示题面中样例的线段树。

翻牌

对应更新线段树，程序中的接口为update_tree(long long* tree, long long* lazy, int node, int start, int end, int L, int R)。这里偷懒地利用递归的方式实现。

以题面给出的测例为例。我们得到的结点集为\(\{0, 1, 2, 4, 5, 6, 7\}\)，于是对应的区间集为\(\{(0,1], (1,2], (2,4], (4,5], (5,6], (6,7]\}\)。事实上，我们（暂时）并不关心每个区间的左、右端点。我们最关心的是其在区间集中对应的rank。例如\((2,4]\)对应[2]，\((4,6]\)对应[3, 4]。当Kidd执行H 2 4的操作时，实际上是对\((1,4]\)执行了一次翻转操作，对应的rank为[1, 2]。

值得注意的是，由于一次操作可能涵盖多个区间，利用线段树的特性，我们可以利用懒惰标记的方式提升效率。在这里，程序以lazy[]数组记录每个结点还未告知其孩子的翻转次数。譬如以第二次翻转操作H 3 6为例，其对应的翻转区间实际上是\((2,6]\)，也即rank中的[2, 4]。在标记\((4,6]\)，也即rank中的[3, 4]时，懒惰标记的策略便将lazy[(4, 6]]++，代表了\((4, 5]\)与\((5, 6]\)由一次未被告知的翻转。

类似于普通线段树中的操作，当遇到一个结点不需再向下查找时，我们直接更新其内容即可。即tree[node] += seg_lenth。若是进行了一次懒惰标记，在更新其本身值的同时还要将lazy[node]++。

查询

对应查询线段树，程序中的接口为query_tree(long long* tree, int node, int start, int end, int L, int R)。这里偷懒地利用递归的方式实现。

过程总结

对于每一次翻牌/查询操作，先利用二分查找找到两个端点在points[]中对应的下标（假设为l，r），那么实际上需要修改的区间即为区间集合中rank为[l, r-1]的区间并。分别调用update_tree与query_tree的接口即可。

复杂度分析

时间复杂度

在读入数据时，总共有\(2m\)个端点，排序去重的复杂度为\(O(mlogm)\)。随后\(m\)次操作，每次操作二分查找花费时间为\(O(logm)\)，修改线段树的时间为\(O(logm)\)，总体为\(O(mlog^2m)\)。对于\(m<200000\)来说，是还可以的复杂度。

空间复杂度

空间开销来源于points[]与线段树segTree[]。由于线段树所需空间的上界不会超过其叶子结点的\(4\)倍，故总的空间复杂度为\(O(m)\)。

其他

关于pushdown：按实现逻辑过程来说，每次query到一个结点就需pushdown一次。在一开始的实现中我确实是如此做的，然而仔细想来，叶子节点的pushdown有可能引发数组越界。不过交到OJ上一直是Wrong Answer——想了很久都想不明白。

直到与高中打OI的同学交流，他们要我把线段树开成\(8\)倍空间试试，这版程序还真过了，但没有报段错误的原因还没找到。直到我想起上学期OOP的某次作业——在堆上开了两个数组，第一个数组越界访问的是第二个数组的空间。在一开始的实现中，我的segTree[]与lazy[]都是分配在堆上的，所以叶子节点pushdown的时候改变的是lazy[]的内容。这便导致了WA而不是段错误。
